<objective>
Build the single shared election helper `bin/lib/coexist.cjs` that every hook will use to
decide run-vs-yield when both plugins are enabled (D-02/D-03). Identity is a pure function
of the resolved script path (`/gsd/` vs `/bm/`), robust when `CLAUDE_PLUGIN_ROOT` is absent
(Pitfall 5). "bm active this session" is a bm self-announced per-session marker keyed on
`session_id` in the GSD temp dir, using a `bm-active-` prefix the 5-minute reaper does NOT
sweep (Pitfall 3). Single-sourcing the election here means a hook added later cannot forget
it (COMPAT-02), and its no-op behavior when no marker exists is what preserves COMPAT-01.

Purpose: The reusable, testable core of hook single-fire election.
Output: coexist.cjs (4 helpers), core.cjs exporting ensureGsdTempDir, and a unit test.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| session_id (from hook stdin JSON) → temp-file path | A CC-provided id is used to compose a marker filename in a shared temp dir |
| shared OS temp dir → marker file | Any local process can read/create files under os.tmpdir()/gsd |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-01 | Tampering | markerPath(sessionId) | mitigate | Validate session_id against `^[A-Za-z0-9_-]+$` before composing the path; malformed → no-op / not-active. Never interpolate a raw id into a path (Task 2, asserted in Task 1) |
| T-14-02 | Tampering | marker file in os.tmpdir()/gsd | mitigate | Marker dir created by ensureGsdTempDir; plain writeFileSync/existsSync (no symlink follow into an attacker path); treat any read failure as "not active" |
| T-14-03 | Spoofing / Elevation | a forged bm-active marker making gsd wrongly yield | accept | Local user-owned temp dir, no privilege boundary crossed. Worst case: the gsd copy yields its stateful checkpoint to the bm copy, which writes the identical format under the same lock (Plan 01); the read-only nudge still emits (Plan 03). No data loss, no escalation |
| T-14-06 | Denial of Service (self) | marker reaped mid-session re-enabling the double-fire | mitigate | `bm-active-` prefix is outside the reaper's default `gsd-` match, AND mtime is refreshed on every bm fire; reaper-safety asserted in Task 1 |
| T-14-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs - Node built-ins only |
</threat_model>
