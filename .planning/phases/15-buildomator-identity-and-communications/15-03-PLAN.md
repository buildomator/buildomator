<objective>
Decouple the generated bm plugin.json description from the gsd source description.
Today stampBmManifest derives the bm description by stripping a literal
"Get Shit Done -- " prefix off the gsd description and prepending "Buildomator -- ".
Once plan 15-02 flips the gsd description to Buildomator-worded, that strip stops
matching and the bm description doubles to "Buildomator -- Buildomator -- ..." (and
can leak a "(legacy /gsd:)" clause the bm package must never show). Replace the
derivation with a fixed authoritative bm description and update the unit test.

Purpose: Protects BRAND-01 in the generated bm package. This is the one [BLOCKING]
code-adjacent edit in the phase; it must land before the dist/bm regen in plan 15-04.
Output: A fixed bm description in stampBmManifest plus a matching drift-test assertion.
</objective>
