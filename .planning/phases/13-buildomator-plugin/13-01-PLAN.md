<objective>
Widen the deterministic bm transform T from "identity stamp only" (Phase 12) to
T = stampBmManifest (now incl. mcpServers rekey) + rewriteCommandRefs + stampHookFallback,
applied inside generate() so build and --check share one code path. Regenerate and commit
dist/bm through the wider transform.

Purpose: makes the generated bm package self-consistent (a bm user's `/gsd:plan-phase`
breadcrumbs become `/bm:plan-phase` instead of invoking the sibling gsd plugin), closes the
known Phase 12 limitation (hooks fallback pointing at the gsd cache dir), and gives the bm
MCP server its own manifest key. Implements D-01, D-02 (refined per RESEARCH F-1), D-03,
D-04, D-05.

D-04 coverage note: the gsd-cache fallback lives in THREE shipped files, not one.
(1) hooks/hooks.json: 17 inline resolvers, slash-form literal. (2) hooks/run-bash-hook.cjs:
its own resolveCandidates() has an independent fallback used to locate the actual hook
script after launch; there the literal is the QUOTED path.join argument pair
'gsd-plugin', 'gsd' (plus one slash form in the header comment). Left unstamped, the bm
plugin under a stale/missing CLAUDE_PLUGIN_ROOT would run hook scripts from the GSD cache
dir, the exact BM-03 failure D-04 prevents, and no hooks.json-only assertion catches it.
(3) bin/check-plugin-update.sh line 16 (PLUGIN_CACHE): a standalone weekly update-notifier
cron, not a Claude Code lifecycle hook, so arguably outside the "hooks fire" scope; it is
stamped anyway for consistency with D-04 intent, and because the stamp makes the bm copy
CORRECT (both packages release from the same repo with aligned versions, so only the cache
dir to inspect differs; REPO and marketplace identifiers stay). All three are stamped by
the same helper and asserted in the drift test and greps below.

Decision refinement to flag for the checker: D-02 said "reuse the skill-name alternation
regex". RESEARCH F-1 measured that the alternation UNDER-rewrites (misses /gsd:capture x35,
/gsd:local-patches x22, /gsd:edit-phase x8, /gsd:extract-learnings whose skill dir is
extract_learnings). The plan uses the anchored leading-slash substitution instead: simpler
AND complete, and it naturally spares gsd:// (38 occurrences, never leading-slash), all
gsd-* filenames, and cache-path literals. This honors D-01's stated goal (bm
self-consistency) and was endorsed as RESEARCH Open Question 1's recommendation.

Output: bin/lib/bm-transform.cjs + unit tests, extended bin/build-bm.cjs, widened
tests/build-bm-drift.test.cjs, regenerated dist/bm.
</objective>
