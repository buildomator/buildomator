# Phase quick-260721-j83 Plan 01: Remove Fable Sunset Time-Gate Summary

Removed the Claude Fable 5 sunset / time-gate from both the CJS and SDK model resolvers so `fable` is now an ordinary always-available tier that resolves straight to `claude-fable-5` regardless of the date.

## What changed

- **bin/lib/core.cjs** — deleted `FABLE_SUNSET_DATE`, `readFableKnob`, `fableAvailable`, `applyFableSunset` and their comments. Renamed the `requestedTier` binding to `tier` so the computed tier flows straight into runtime resolution; removed the four names from `module.exports`. Kept `'fable'` in `VALID_TIERS`.
- **sdk/src/query/config-query.ts** — mirrored the removal (same four symbols plus the `GSD_FABLE_SUNSET_NOW` read). Both call sites now pass the tier through unchanged. Dropped the now-unused `readFileSync` import.
- **bin/lib/config-schema.cjs / sdk/src/query/config-schema.ts** — deleted the `'fable.mode'` and `'fable.until'` keys from both key sets (parity preserved).
- **tests/fable-sunset.test.cjs** — deleted (it only exercised the removed gate); no empty replacement left behind. `tests/fable-tier.test.cjs` untouched.
- **README.md** — reworded the fable tier row to describe it as a first-class model tier (Anthropic runtimes get `claude-fable-5`, non-Claude runtimes alias to their top model); dropped the sunset-window / auto-fallback / `fable.mode`/`fable.until` prose.
- **sdk/dist** rebuilt and **dist/bm** regenerated so the live spawn path and the bm twin carry no gate.

`sdk/shared/model-catalog.json`, `model-profiles.cjs`, and historical CHANGELOG entries were not touched. No version bump, no CHANGELOG entry.
