# Quick Task 260629-35t — Summary

Exploration (no code change). Produced a grounded design for integrating
evidence-based verification into GSD with low token overhead.

## Deliverable

`260629-35t-EXPLORATION.md` — diagnosis, current-machinery map, design, ranked
integration seams, and an incremental rollout.

## Key conclusions

- **Gap:** `gsd-verifier` is static by design ("do not start the app"); UAT asks
  the user; the executor's `<verify>` is pass/fail unit-test shaped. Nothing
  reproduces the original symptom and observes it gone — so "fixed" is claimed
  from code-shaped evidence.
- **Fix:** a red→green evidence loop (capture failing repro → fix → capture
  passing repro, cite both by path), highest-value in `gsd-debug`.
- **Low overhead:** capture inside subagents (zero per-turn cost); CLI over
  always-on Playwright-MCP (~20k tokens/turn); evidence saved to
  `.planning/evidence/` as files referenced by path, never read into context
  unless a check fails; delegate to the project's run/observe skill; one
  `workflow.evidence_verification` gate (code_review pattern).
- **Rollout:** Step 1 = convention + gate + wire into `gsd-debug` (small,
  standalone, non-disruptive to v4.1). Steps 2-3 = executor `capture:` +
  verifier evidence-mode + generalized verify-work (a later phase/milestone).
